# 📄 ОТЧЕТ ПО КУРСОВОМУ ПРОЕКТУ: «**SPA** «Менеджер задач (**To-Do+**) виртуальных испытаний» »
##  Цель проекта
**Инженерно-технологическая цель:**  
Разработка и внедрение кроссплатформенной веб-среды диспетчеризации «To-Do+» для Консалтингового центра цифрового моделирования, обеспечивающей сквозной мониторинг жизненного цикла и верификацию результатов конечно-элементных (FEM) и гидрогазодинамических (CAE/CFD) виртуальных испытаний изделий машиностроения. Система предназначена для автоматизации учета расчетных задач, исключения влияния человеческого фактора при первичной оценке результатов ОТК и предоставления руководству предиктивной аналитики по качеству проектируемых конструкций (изделий).

**Учебно-академическая цель:**  
Практическое применение и демонстрация компетенций в области проектирования современных одностраничных веб-приложений (SPA) на базе библиотеки React.js с использованием компонентного подхода, декларативного управления состоянием (Hooks, Context API), интеграции с браузерными интерфейсами (BOM API), отказоустойчивого локального хранения данных (Web Storage) и обеспечения адаптивности под мобильные типы ПЭВМ в соответствии с промышленными стандартами Clean Code и Industry 4.0.

## ⚙️ 2. Спецификация бизнес-логики и ТЗ (CRUD)

В соответствии с требованиями к полноценным CRUD-системам реализован полный цикл управления данными:
* **Create (Создание)**: Регистрация новой заявки на расчет через форму `AddTaskForm.jsx`. При создании объект получает уникальный неизменяемый идентификатор `crypto.randomUUID()` (защита от коллизий) и технологический буквенно-цифровой шифр `GEO-XXXX-XX`.
* **Read (Чтение)**: Потоковое считывание массива задач из `localStorage` при старте с try/catch обработкой на случай повреждения JSON-структуры.
* **Update (Обновление)**: Интерактивное переключение стадий выполнения задачи («Новая», «В расчете», «Верификация», «Завершено») через выпадающие селекторы.
* **Delete (Удаление)**: Мгновенное исключение записи из реестра методом фильтрации массива `.filter()` без использования устаревших блокирующих диалоговых окон.

## 🛠️ 3. Инженерное ядро и реальные математические расчеты

Для исключения «заглушек» в папку `/src/utils/cartCalc.js` внедрен полноценный физико-математический процессор:
1. **Коррекция погрешности (Mesh Convergence)**: Программа считывает плотность конечно-элементной сетки (`lod`) и домножает базовое тензометрическое значение на коэффициент точности (High LOD = `1.02`, Low LOD = `1.35`).
2. **Вычисление по дисциплинам CAE**:
   * *Механика (Прочность)*: Рассчитывает коэффициент запаса прочности относительно предела текучести стали ($limit = 250$ МПа).
   * *Теплофизика*: Рассчитывает запас теплостойкости относительно порога термической деформации сплава ($limit = 600$ °C).
   * *CFD (Гидродинамика)*: Вычисляет гидродинамическую стабильность потока ($limit = 40$ м/с).
3. **Автоматический контроль качества (ОТК)**: Нормативный критерий прохождения теста — коэффициент запаса $\geq 1.2$. Если деталь не выдерживает нагрузку, утилита аппаратно присваивает ей статус дефекта (`failed`). Строка в реестре автоматически окрашивается в светло-красный цвет, сигнализируя о браке конструкции.

## 🛡️ 5. Распределение прав доступа (RBAC-модель)

В приложении аппаратно разграничены рабочие пространства двух ключевых ролей Консалтингового центра и Заказчика-компании:
* **Consultant (Консультант КЦ)**: Обладает правами администратора. Ему доступна форма создания задач, управление выпадающими селекторами стадий и кнопки полного удаления записей `✕`.
* **Engineer (Инженер Заказчика-компании)**: Обладает правами исполнителя с доступом Read-Only. Форма регистрации скрывается с экрана, кнопки удаления полностью вырезаются из DOM, а селектор статусов переводится в режим `disabled`, защищая реестр от несанкционированного изменения данных.

## 📱 6. Мобильная адаптивность под любые устройства

Интерфейс спроектирован по методологии адаптивного дизайна для ПЭВМ, планшетов и смартфонов:
* **Трансформация таблиц**: При ширине экрана менее `1024px` громоздкая инженерная таблица реестра автоматически реструктуризируется. `<thead>` скрывается, а каждая строка `<tr>` превращается в обособленную карточку изделия.
* **CSS-магия `data-label`**: Подписи полей на мобильных устройствах генерируются динамически через системный CSS-атрибут `content: attr(data-label)`, исключая дублирование HTML-кода и гарантируя отсутствие горизонтальной прокрутки.

## ⚡ 7. Оптимизация и интеграция с BOM API

* **Синхронизация вкладок браузера**: Настроен динамический хук управления элементами `<head>`. При переводе задачи в стадию расчета вкладка запускает анимированный индикатор (`.gif`), а мобильная строка состояния (`theme-color`) плавно окрашивается в синий цвет. При успехе или браке вкладка моментально выводит зеленую галочку или красный крестик.
* **Защита от утечек памяти (Memory Leaks)**: Все BOM-слушатели событий сети (`online` / `offline`) снабжены функцией возврата очистки (`cleanup function`) внутри `useEffect`, исключая избыточную нагрузку на CPU.
* **Предотвращение холостых ререндеров**: Использование `useCallback` для функций CRUD и `useMemo` для фильтрации данных снижает вычислительную нагрузку на дерево компонентов до минимума.
